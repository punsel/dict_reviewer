Object-Oriented Programming

1. What is Object-Oriented Programming (OOP)?

Answer: OOP is a programming paradigm based on objects that contain data (attributes) and methods (behavior).

2. What is a Class?

Answer: A class is a blueprint/template for creating objects. It defines the structure and behavior that the objects will have.

3. What is an Object?

Answer: An object is an instance of a class. It has its own state and can perform behaviors defined by the class.

4. Define Encapsulation.

Answer: Encapsulation is the process of hiding data inside the class and providing controlled access using getters and setters.

Keyword: private fields, public methods.

5. What is Abstraction?

Answer: Abstraction is hiding unnecessary implementation details and showing only the essential features.

Example: You use a phone without knowing the circuit design.

6. What is Inheritance?

Answer: Inheritance allows one class (child) to acquire attributes and behaviors of another class (parent).

Syntax Example (Java): class Dog extends Animal { }

7. What is Polymorphism?

Answer: Polymorphism means one function name behaving differently depending on the context.

  • Method Overloading (same method name, different parameters)
  • Method Overriding (child class modifies parent method)

8. Define Method Overloading.

Answer: Same method name, different parameter lists, occurs in the same class.

9. Define Method Overriding.

Answer: A child class redefines a method inherited from its parent class.

10. What is a Constructor?

Answer: A special method used to initialize objects. It has the same name as the class and no return type.

11. What is a Default Constructor?

Answer: A constructor with no parameters, automatically provided if no constructors are defined.

12. What is the this keyword used for?

Answer: To refer to the current object instance, often used to distinguish class fields from local variables.

13. What is the super keyword used for?

Answer: To refer to the parent class, often used when calling parent class constructors or methods.

14. What is an Interface?

Answer: An interface is a contract that contains abstract methods only.
Classes must implement it and provide the method bodies.
Key Word: implements.

15. What is an Abstract Class?

Answer: A class that cannot be instantiated and may contain both abstract and non-abstract methods.
Key Word: abstract.

17. What is Exception Handling?

Answer: A mechanism to handle runtime errors using keywords: try, catch, finally, throw, throws.

18. What does try-catch-finally mean?

  • try → Code that may cause an exception
  • catch → Handles the error
  • finally → Always executes (even if error occurred)

19. What is UML in OOP?

Answer: UML (Unified Modeling Language) is used to visually model classes, relationships, and interactions before implementation.
