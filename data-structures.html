Data Structures

1. What is a Data Structure?

Answer: A data structure is a method of organizing and storing data so that it can be accessed and modified efficiently.

Key Idea: Different tasks require different data structures to optimize speed and memory.

2. What is the difference between Primitive and Non-Primitive Data Structures?

Answer:

Primitive: Basic built-in types (int, char, float).

Non-Primitive: Organized collections (arrays, lists, stacks, queues, trees).

Key Idea: Non-primitive structures are used to store larger and complex collections of data.

3. What is an Array?

Answer: An array is a collection of elements stored in contiguous memory locations, accessed using an index.

Time Complexity: Access = O(1).

4. Why is array insertion costly?

Answer: Because inserting requires shifting elements, resulting in O(n) time complexity.

5. What is a Linked List?

Answer: A linked list is a linear structure where each element (node) stores data and a reference to the next node.

Advantage: Fast insert/delete = O(1) when position is known.

6. What is the drawback of Linked Lists compared to Arrays?

Answer: No direct indexing; accessing an element requires traversal (O(n)).

7. Differentiate Singly and Doubly Linked Lists.

Answer:

Singly: Each node has one pointer (next).

Doubly: Each node has two pointers (next and prev) → allows two-way traversal.

8. What is a Stack?

Answer: A stack is a LIFO (Last In, First Out) data structure.

Examples: Undo operations, function call stack.

9. What are the primary operations in a Stack?

Answer:

push() → insert
pop() → remove
peek() → view top element

10. What is a Queue?

Answer: A queue is a FIFO (First In, First Out) structure.

Example: Scheduling tasks, printer jobs.

11. What is a Circular Queue?

Answer: A queue where the last position connects back to the first to reduce wasted space.

12. What is a Binary Tree?

Answer: A hierarchical data structure where each node has at most two children.

13. What is a Binary Search Tree (BST)?

Answer: A binary tree where:

Left subtree < Root < Right subtree

Search Time: Best = O(log n), Worst (skewed) = O(n).

14. What causes a BST to become unbalanced?

Answer: Inserting elements in sorted order, causing it to resemble a linked list, increasing search time.

15. What is Tree Traversal? Name types.

Answer: Visiting all nodes.

In-order (L, Root, R)
Pre-order (Root, L, R)
Post-order (L, R, Root)

16. What is a Hash Table?

Answer: A structure that stores data in key-value pairs using a hash function.

Search Time: Average = O(1).

17. What is a Hash Collision and how is it resolved?

Answer: When two keys hash to the same index.

Solution: Chaining (linked list per bucket) or Open Addressing.

18. What is Recursion commonly used for in Data Structures?

Answer: Tree and graph traversal, divide-and-conquer algorithms like QuickSort.

19. Define Time Complexity.

Answer: A measure of how execution time grows with input size n.

Examples: O(1), O(log n), O(n), O(n²)

20. Which data structure is best for implementing Undo/Redo operations and why?

Answer: Stack, because it follows LIFO — last action undone first.
