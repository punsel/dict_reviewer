SDLC

1. What does SDLC stand for?

Answer: Systems Development Life Cycle

Framework that describes the stages of system development.

2. Which SDLC phase focuses on understanding user needs and system goals?

Answer: Requirement Analysis

Interviews, surveys, observation, requirement documentation occur here.

3. Which phase is responsible for designing system architecture, database models, and UI layouts?

Answer: System Design

Converts requirements into blueprint.

4. In which phase is the actual coding done?

Answer: Implementation / Development Phase

Programmers build the system based on the design specs.

5. What is the goal of the Testing phase?

Answer: To detect errors/bugs and verify the system meets requirements.

Ensures correctness and quality.

6. What happens in the Deployment phase?

Answer: The system is installed, released, or made available to end-users.

7. The phase where updates, patches, and user support occur is called what?

Answer: Maintenance

After release, the system is improved and fixed as needed.

8. Name the SDLC model where processes are done step-by-step and cannot go back to previous stages.

Answer: Waterfall Model

Linear and strict.

9. Name the SDLC model where development is done in repeated cycles (sprints).

Answer: Agile Model

Flexible, iterative, user feedback-driven.

10. Which SDLC model is best when requirements are not yet clear at the start?

Answer: Agile

Agile allows changing requirements.

11. Which SDLC model is best when requirements are very clear and stable?

Answer: Waterfall

Works well when everything is defined upfront.

12. What is the main advantage of Agile?

Answer: Continuous user involvement and quick adaptation to changes.

13. What is the main disadvantage of Waterfall?

Answer: Difficult to go back to previous phases; no early working product.

14. What is a deliverable of the Requirements Phase?

Answer: SRS (Software Requirements Specification)

Contains functional and non-functional requirements.

15. What is a deliverable of the System Design Phase?

Answer: System Architecture / ERD / Class Diagrams / UI Mockups

17. Who acts as the voice of the client in Agile?

Answer: Product Owner

Prioritizes features and backlog.

18. What document lists prioritized features or tasks in Agile?

Answer: Product Backlog

19. Which Agile role ensures the team follows Agile principles and resolves blockers?

Answer: Scrum Master

20. Which approach involves delivering software in small functional increments?

Answer: Iterative / Incremental Development

Builds the system piece-by-piece.
